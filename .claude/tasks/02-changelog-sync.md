## Goal
Auto-sync `CHANGELOG.md` into the VitePress docs site so the changelog page always shows the full, up-to-date release history without any manual maintenance.

## Context
- `CHANGELOG.md` — auto-generated by semantic-release (root level)
- `website/changelog.md` — currently a manual summary page (out of date)
- VitePress supports `<!--@include: ../CHANGELOG.md-->` for file includes
- `.github/workflows/docs.yml` — deploys docs but doesn't watch CHANGELOG.md changes

## Steps
1. Rewrite `website/changelog.md` to:
   - Keep a short intro paragraph explaining it's auto-generated
   - Use VitePress include syntax to embed the real `CHANGELOG.md`
2. Add `CHANGELOG.md` to the `push` trigger paths in `.github/workflows/docs.yml`
3. Create `scripts/changelog-preview.mjs` — a script that runs `npx semantic-release --dry-run` and prints what the next changelog entry would look like (for local developer use)
4. Add `"changelog:preview": "node scripts/changelog-preview.mjs"` to `package.json` scripts

## Acceptance criteria
- [ ] `website/changelog.md` uses VitePress `<!--@include: ../CHANGELOG.md-->` include
- [ ] `docs.yml` triggers on `CHANGELOG.md` changes
- [ ] `scripts/changelog-preview.mjs` exists and is documented
- [ ] `npm run changelog:preview` works locally
- [ ] Status: DONE ✅
